Задачей scp-программы формирования терминологической онтологии для заданной предметной области является создание терминологической онтологии для заданной предметной области. Первым входным параметром является предметная область, для которой будет генерироваться терминологическая онтология. Вторым входным параметром является узел ответа, куда будут занесены все сгенерированные конструкции. В ходе выполнения программы явно указывается связь со следующими ключевыми узлами: множествами предметных областей и терминологических онтологий, отношениями онтология*, основной идентификатор*, идентификатор*, а также подмножества множества естественных языков, на которых представлены идентификаторы и основные идентификаторы понятий, принадлежащих терминологической онтологии.

В случае, когда в качестве аргумента подана не предметная область, генерируется комментарий на английском языке, сообщающий о том, что на вход подан аргумент, который не является предметной областью. В случае, когда на вход подана пустая предметная область, то есть предметная область, для которой не указан ни один ключевой элемент, генерируется комментарий на английском языке, сообщающий, что в поданной на вход предметной области отсутствуют ключевые элементы.
